What is JavaScript?

JavaScript is a cross-platform, object-oriented scripting language used to make webpages interactive (ex. having complex animations, clickable buttons, popup menus, etc.). There are also more advanced server side versions of JavaScript such as Node.Js which allow you to add more functionality to a website...

Mozilla
  • Developed in 10 days by Brendan Eich in 1995
  • Has nothing to do with the Java language - its naming is a marketing ploy
  • First appeared in Netscape Navigator (the predecessor of Firefox)
  • Standardized and monitored by the European ECMA (referred to as ECMAScript or ES)
>

Examples of JavaScript usage:

  • Web forms processing

    • Input validation
    • Form controls
    • Button actions

  • Web pages animations and transitions

    • Menus and navigation
    • Drag and drop
    • Animations and transitions (i.e. this presentation)

  • Phone apps

    Many phone apps are written in JavaScript to allow coding for both Android and iOS

  • Server side

    Many web sites run JavaScript on the server, using the same language in the front and back ends

JavaScript main features

  • High level (abstracts from the details of the computer or environment)
  • Interpreted (rather then compiled)
  • Object oriented (more about that in week 3)
  • Multi-platform (can run on any machine, OS, or device - both on the client and server side)
  • Most features of the language work in every browser for exceptions check CanIUse

JavaScript Interpreter

Most applications we use (such as this browser) are compiled, meaning a special program called compiler processed the source code into machine language.
While compiled applications are (generally speaking) faster, they are tied to a specific platform (OS, or computer), are usually large files, and are hard to debug once compiled.

An interpreter executes each line of code as it reads it. This allows us to use the same source code for many platforms (in our case, browsers), the source code is relatively small (and can be further minimized and compressed), and is much easier to debug in run-time.
Your browser has a JavaScript engine, that acts as an interpreter.

Common JavaScript engines:

  • V8 - Chrome/Chromium-based browsers/Node JS/Android
  • SpiderMonkey - Firefox
  • Chakra - Microsoft Edge
  • JavaScriptCore - Safari/WebKit based browsers/iOS

The REPL

JavaScript is an interpreted programming language, meaning the JavaScript engine in your browser takes one instruction at a time, interprets it into bytecodes, and executes it. That makes the language dynamic and fast, and allows for all kinds of optimizations.

JavaScript provides a REPL (Read-Eval-Print Loop) that allows us to freely type JavaScript instructions and have them executed right away.

You have a REPL like that in your browser, in your Developer Tools. Let's access it:

  • In Chrome, go to Menu > More Tools > Developer Tools
    or click CTRL+SHIFT+I (CMD+SHIFT+I on Mac)
    or right-click the mouse anywhere on the page, and select Inspect
  • In Firefox go to menu > Web Develop >Toggle Tools
    or click CTRL+SHIFT+I (CMD+SHIFT+I on Mac) or right-click the mouse anywhere on the page, and select Inspect Element
  • In Microsoft Edge select Developer Tools from the menu, or click F12
    or right-click the mouse anywhere on the page, and select Inspect
  • In Safari Choose Safari > Preferences, and click Advanced
    At the bottom of the pane, select the “Show Develop menu in menu bar” checkbox
    Choose Develop > Show Web Inspector

In all browsers, you'll be able to switch to a tab named Console
In Chrome, you may need to switch to the Sources tab, before Console becomes available.

Things you can type into the REPL (hit Enter to start evaluation):

  • Numbers
  • Numeric operations (try typing 1+1 or 2*5)
  • Strings - a set of characters surrounded by quotes or double quotes
  • Console functions - try typing console.log('hello')
  • Functions - try typing Math.sqrt(4)
  • Assignments - try typing a = 'my name'

JavaScript Expressions

Expressions are units of code that can be evaluated to a value.

  • Numbers - 2, -7, 4.1
  • Arithmetic expressions - 1 + 1, 75 / 3, 2 + 4 + 5
  • Logical expressions - 7 > 2, 8 < 10 && 1 < 2
  • String expressions - 'something' or "something"
  • We'll encounter more expression types later on when we discuss array, objects and functions

Numbers

Types of numbers that can be used in JavaScript:

  • Integers - whole numbers between -231 to 231
  • Floating point - fractions, numbers containing a '.'
  • Numbers in different bases:
    • Binary - 0b1110 (0b or 0B and the digits 0,1)
    • Octal - 0o123 (0o or 0O and the digits 0-7)
    • Hexadecimal - 0x1A3 (0x or 0X, the digits 0-9 and the letters A-F)
  • NaN - not a number - the result of trying to evaluate something that is not a legal number (like Math.sqrt(-1))

Arithmetic Expressions

Numbers and arithmetic operators between them. that evaluate to numbers. Available operators:
  • + - addition
  • - - subtraction
  • * - multiplication
  • / - division
  • % - modulus (division remainder)
  • ** - power
Remember your operators precedence!
  • All operators (other than **) are evaluated left-to-right
  • *, /, ** and % precede + and -
  • You can control the precedence by using parentheses

Logical Expressions

Logical expressions contain statements that evaluate to true or false.
Another feature of JavaScript, owing to it's C roots - false gets evaluated to zero and true to non-zero (usually 1)
and vice versa.
  • > - greater than
  • < - less than
  • ==, === - equal to, strict equal to
  • !=, !== - not equal to, strict not equal to
  • ! - logical NOT
  • && - logical AND
  • || - logical OR
Remember your operators precedence!
  • Operators are evaluated left to right
  • !, || and && precede equality tests
  • arithmetic expressions precede both!
  • You can control the precedence by using parentheses

  • true || false

Strings

In JavaScript, strings are a series of characters enclosed in quotes (') or double-quotes (")
Characters can be any Unicode character, including spaces, tabs etc:
"中文 español Deutsch English देवनागरी العربية português বাংলা русский 日本語 ਪੰਜਾਬੀ 한국어 தமிழ் עברית"

Things we can do with strings:

  • We can compare them: 'Guy' !== 'guy'
  • We can concatenate them using the '+' sign: 'hello + ' world'
  • To include special characters in a string, you need to use escape characters (examples: \n - newline, \t - tab, \" - quotes
  • Strings are 0-based (the first letter is in place 0, the second 1 etc.), so 'hello'[1] === 'e'
  • You can extract numbers from strings using the functions parseInt and parseFloat
Even though both quotes and double-quotes are legal in JavaScript, we'll use quotes (') for our strings,
to prevent confusion with HTML attributes, that must use "":
'<div class="myClass">' + 'my content' + '</div>'
